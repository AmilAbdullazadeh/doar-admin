# Alerts

<p class="sbdocs-desc">
    Provide contextual feedback messages for typical user actions.
</p>

<Canvas>
    <Story
        name="Alert Controls"
        argTypes={{
            variant: {
                control: {
                    type: `select`,
                },
            },
            color: {
                control: {
                    type: `select`,
                },
            },
        }}
        args={{
            variant: `contained`,
            color: `primary`,
            solid: false,
            hasLink: false,
            isDismissible: false,
            hasIcon: false,
            children: `A simple alert—check it out!`,
        }}
    >
        {(args) => {
            const { children, hasLink, hasIcon, ...restArgs } = args;
            if (hasLink) {
                return (
                    <Alert hasLink={hasLink} hasIcon={hasIcon} {...restArgs}>
                        A simple primary alert with{" "}
                        <a href="/">an example link.</a> Give it a click if you
                        like.
                    </Alert>
                );
            } else if (hasIcon) {
                return (
                    <Alert hasLink={hasLink} hasIcon={hasIcon} {...restArgs}>
                        <AlertCircle size="24" /> A primary alert—check it out!
                    </Alert>
                );
            } else {
                return (
                    <Alert hasLink={hasLink} hasIcon={hasIcon} {...restArgs}>
                        {children}
                    </Alert>
                );
            }
        }}
    </Story>
</Canvas>

## Default Functionality

Alerts are available for any length of text, as well as an optional dismiss button. For proper styling, use one of the eight required contextual props.

<Preview withToolbar>
    <Story name="Color">

## Alert Link

Use the `hasLink` prop to quickly provide matching colored links within any alert.

<Preview withToolbar>
    <Story name="Has Link">
        <Alert color="primary" hasLink>
            A simple primary alert with <a href="/">an example link.</a> Give it
            a click if you like.
        </Alert>
    </Story>
</Preview>

## Additional Content

Alerts can also contain additional HTML elements like headings, paragraphs and dividers.

<Preview withToolbar>
    <Story name="Additional Content">
        <Alert color="success">
            <h6>Well done!</h6>
            <p>
                Aww yeah, you successfully read this important alert message.
                This example text is going to run a bit longer so that you can
                see how spacing within an alert works with this kind of content.
            </p>
            <hr />
            <p>
                Whenever you need to, be sure to use margin utilities to keep
                things nice and tidy.
            </p>
        </Alert>
    </Story>
</Preview>

## Dismissing

it’s possible to dismiss any alert inline. Pass `isDismissible` to add a dismiss button, which adds extra padding to the right of the alert and positions the .close button.

<Preview withToolbar>
    <Story name="Dismissing">
        <Alert color="warning" isDismissible>
            <strong>Holy guacamole!</strong> You should check in on some of
            those fields below.
        </Alert>
    </Story>
</Preview>
